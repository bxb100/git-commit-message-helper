Git Commit Message Helper helps you write cleaner, more consistent commit messages without leaving the IDE.

Git Commit Message Helper 帮助你在 IDE 内更高效地编写清晰、统一、规范的提交信息。

It combines a structured commit editor, customizable Conventional Commit style templates, and provider-aware LLM assistance in one IntelliJ plugin. You can create commit messages manually, generate them from selected changes, reformat existing drafts, and tune the final output to match your team's workflow.

它将结构化提交编辑器、可自定义的 Conventional Commit 风格模板，以及支持多 Provider 的 LLM 辅助能力整合到同一个 IntelliJ 插件中。你既可以手动编写提交信息，也可以根据所选变更生成草稿、重写已有内容，并按团队规范输出最终结果。

What It Does

  • Adds commit message actions directly to the IntelliJ VCS commit panel.
  • Helps compose commit messages with structured fields such as type, scope, subject, body, breaking change, closes, and skip ci.
  • Supports customizable commit templates powered by Apache Velocity.
  • Lets you manage allowed commit types and their descriptions.
  • Generates commit messages from the selected git changes with an LLM.
  • Rewrites existing commit messages to match the configured template.
  • Parses an existing message back into structured fields with Smart Echo.
  • Supports skip-CI presets, configurable field visibility, and action visibility.
  • Includes localized resources for English, Chinese, Japanese, and Korean.
  • 在 IntelliJ 的 VCS 提交面板中直接提供提交信息辅助动作。
  • 通过结构化字段组织提交信息，包括 type、scope、subject、body、breaking change、closes 和 skip ci。
  • 支持基于 Apache Velocity 的自定义提交模板。
  • 支持维护可用的提交类型及其描述。
  • 支持基于当前选中 Git 变更通过 LLM 生成提交信息。
  • 支持将已有提交信息按当前模板重新格式化。
  • 支持通过 Smart Echo 将现有提交信息回填为结构化字段。
  • 支持 Skip CI 预设、字段显隐以及动作可见性配置。
  • 内置英文、中文、日文、韩文本地化资源。

Main Actions

  • Generate Commit Message: create a draft from the currently selected git changes.
  • Format Commit Message: rewrite the current commit message using your configured template.
  • Create Commit Message: open the structured editor for manual commit composition.
  • Generate Commit Message：根据当前选中的 Git 变更生成提交信息草稿。
  • Format Commit Message：按照你配置的模板重写当前提交信息。
  • Create Commit Message：打开结构化编辑器，手动编写提交信息。

Each action can be shown or hidden independently, so teams can expose only the workflow they actually use.

这 3 个动作都可以单独控制是否显示，方便团队只保留实际使用的工作流入口。

Structured Commit Editing

The manual editor is designed for teams that want strict control over commit structure. Instead of editing a single plain text box, you can fill dedicated fields for the header, body, breaking changes, issue links, and optional skip-CI markers.

结构化编辑器适合对提交格式要求较高的团队使用。你不需要只面对一个纯文本输入框，而是可以分别填写标题、正文、 BREAKING CHANGE、关联 issue 以及可选的 Skip CI 标记等字段。

This makes commit messages easier to review, easier to standardize, and easier to adapt when your team uses a formal commit convention.

这种方式让提交信息更易审阅、更易统一，也更适合需要遵循正式提交规范的团队。

Default Commit Style

The plugin is ready to use out of the box with a Conventional Commit style template such as:

插件开箱即用，默认提供类似 Conventional Commits 的提交模板，例如：

type(scope): subject

body

BREAKING CHANGE: changes

Closes issue

[skip ci]

The built-in default types include feat, fix, docs, style, refactor, perf, test, build, ci, chore, and revert.

插件默认内置的提交类型包括 feat、fix、docs、style、refactor、perf、test、build、ci、chore 和 revert。

The template text, visible fields, type list, display style, and skip-CI behavior are all configurable.

模板内容、字段显示、类型列表、展示方式以及 Skip CI 行为都可以按团队需要调整。

LLM Workflow

The plugin can use an LLM to generate commit messages from the selected changes, reformat an existing draft, or parse a free-form message back into structured fields. This reduces manual editing while still keeping the final result under your control.

插件可以通过 LLM 根据选中变更生成提交信息、重写已有草稿，或将自由文本提交信息重新解析为结构化字段。这样既能减少手工编辑， 也能保证最终结果仍然受你控制。

Smart Echo

When Smart Echo is enabled and the commit panel already contains text, the manual editor can ask the LLM to map that text back into fields like type, scope, subject, body, changes, closes, and skipCi.

当开启 Smart Echo 且提交面板中已经存在文本时，手动编辑器可以调用 LLM，把当前内容回填为 type、scope、subject、body、 changes、closes 和 skipCi 等结构化字段。

Provider Compatibility

The plugin currently supports two provider modes: OpenAI Compatible and Anthropic. This lets teams connect either official endpoints or compatible internal gateways, depending on their environment.

插件当前支持两种 Provider 模式：OpenAI Compatible 和 Anthropic。这让团队可以根据 实际环境接入官方接口或兼容的内部网关。

Configuration

Open File | Settings | GitCommitMessageHelper to configure the plugin. The settings are split into focused pages so you can tune both the authoring workflow and the LLM integration.

打开 File | Settings | GitCommitMessageHelper 即可配置插件。设置项按照不同功能拆分，便于分别调整提交编写流程和 LLM 集成能力。

General Settings

Choose how commit types are displayed, how many appear inline, which fields are hidden, whether skip-CI options are enabled, and which actions should be visible in the commit panel.

你可以设置提交类型的展示方式、内联显示数量、隐藏哪些字段、是否启用 Skip CI 选项，以及在提交面板中显示哪些动作。

Commit Template

Customize the final commit message with an Apache Velocity template so the generated output matches your team's exact format.

你可以通过 Apache Velocity 模板自定义最终提交信息的渲染格式，让输出结果严格符合团队规范。

Commit Types

Edit the allowed commit types and their descriptions to align with your workflow, terminology, and review conventions.

你可以自由维护允许使用的提交类型及其描述，使之与团队术语、工作流和评审习惯保持一致。

LLM Settings

Configure the provider, base URL, API key, model, temperature, response language, and Smart Echo behavior.

你可以配置 Provider、Base URL、API Key、模型、Temperature、返回语言以及 Smart Echo 行为。

For OpenAI-compatible services, the plugin can work with either a full /chat/completions endpoint or a base API URL. For Anthropic, it uses the native Messages API.

对于 OpenAI Compatible 服务，插件既支持直接填写完整的 /chat/completions 接口，也支持填写基础 API 地址。 对于 Anthropic，则会使用其原生 Messages API。

Project Background

This project started as an enhanced version of git-commit-template and has grown into a more configurable commit authoring assistant for IntelliJ-based IDEs.

这个项目最初基于 git-commit-template 增强而来，当前已经演进为一个面向 IntelliJ 平台、可配置能力更完整的提交信息辅助插件。
